# ChatGPT History Jump 中文说明

`ChatGPT History Jump` 是一个面向 ChatGPT 网页端的浏览器扩展，用来在会话页面右侧生成“历史问题目录”，帮助你快速定位自己在当前对话中提过的问题。

## 功能介绍

- 自动扫描当前会话中的用户提问
- 在页面右侧生成可搜索的问题目录
- 点击目录项后跳转到对应提问位置
- 自动高亮当前视口附近的问题
- 支持记住每个会话的目录折叠状态
- 支持识别带图片的问题，并显示“含图”标记
- 支持长问题折叠显示，并通过 `+` / `-` 按钮展开或收起
- 支持本地设置面板位置、宽度、显示密度和主题
- 回复标题默认只显示最高一级，次级标题可在对应一级标题下展开
- 支持对超长对话手动深度扫描，主动滚动加载更早的问题
- 页面加载或切换对话后自动深度扫描（可在设置中关闭）
- 回复标题改为懒解析，优先保证超长对话的问题索引速度
- 切换对话时清空并隔离问题/标题缓存，避免显示上一条对话内容
- 点击未加载的问题时会尝试滚动找回对应消息并跳转
- 深度扫描后按页面发现顺序重新排序缓存问题
- 回复标题会合并语义标题和编号/视觉一级标题，避免只显示前几条
- 扫描后会后台预解析少量已加载回复标题，兼顾速度和可见性

## 适用页面

当前扩展会注入到以下域名：

- `https://chatgpt.com/*`
- `https://chat.openai.com/*`

## 安装方式

在 Chrome Web Store 页面发布前，可以先通过本地加载方式在 Chrome 和 Edge 中手动安装。

### Chrome 安装步骤

1. 打开 Chrome 浏览器，进入 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择当前项目目录
5. 打开 ChatGPT 会话页面，刷新后即可看到右侧目录面板

### Edge 安装步骤

1. 打开 Edge 浏览器，进入 `edge://extensions/`
2. 打开左下角“开发人员模式”
3. 点击“加载解压缩的扩展”
4. 选择当前项目目录
5. 打开 ChatGPT 会话页面，刷新后即可看到右侧目录面板

## 使用说明

### 1. 查看问题目录

进入任意 ChatGPT 会话后，扩展会自动扫描当前页面中的用户消息，并在右侧显示问题列表。

### 2. 搜索问题

在侧边栏顶部的搜索框中输入关键词，可以快速过滤当前会话中的问题。

### 3. 快速跳转

点击任意目录项，会平滑滚动到页面中对应的问题位置，并闪烁提示目标区域。

### 4. 查看带图问题

如果某条用户问题中包含图片，目录项会显示：

- `含图`
- `含N图`

这样可以更快识别带附件或截图的提问。

### 5. 展开长问题

如果问题内容较长，目录里默认只展示精简内容。点击右侧的 `+` 按钮可以展开完整文本，再次点击 `-` 可收起。

### 6. 折叠目录

点击目录左侧的小按钮可以折叠或展开整个侧边栏。扩展会按当前会话地址保存折叠状态。

### 7. 自定义显示

点击标题栏中的设置按钮，可以选择目录显示在左侧或右侧，并调整宽度、显示密度和主题。设置会保存在浏览器本地。

### 8. 查看回复标题层级

展开回复标题时，目录默认只显示当前回复中的最高一级标题。如果某个一级标题下还有次级标题，可以点击该标题右侧的按钮展开查看。

### 9. 自动深度扫描

扩展会默认在页面加载和切换对话后自动执行深度扫描，尽可能多地索引历史消息。扫描过程中会显示进度百分比，扫描完成后自动恢复到原来的阅读位置。如果不需要自动扫描，可以在设置面板中关闭"自动扫描"选项。

### 10. 手动深度扫描超长对话

如果自动扫描未覆盖全部内容，也可以点击标题栏中的深度扫描按钮手动触发。

### 10. 跳转未加载问题

如果某条缓存问题显示“未加载”，说明对应消息暂时不在当前页面 DOM 中。点击该条目后，扩展会尝试上下滚动找回该消息，找到后自动跳转。

## 实现原理

扩展当前基于 `content script` 工作，主要逻辑如下：

- 监听页面中的用户消息节点
- 提取用户问题文本
- 判断问题是否带图片
- 建立问题列表与页面节点的映射关系
- 通过 `MutationObserver` 响应页面内容变化
- 通过 `IntersectionObserver` 同步当前高亮项
- 通过 `chrome.storage.local` 保存折叠状态和本地显示设置
- 深度扫描会临时滚动当前会话，以触发 ChatGPT 加载更早的 DOM 节点
- 回复标题只在用户展开标题预览时解析，避免普通扫描反复遍历回复 DOM
- 对话缓存按当前会话路径隔离，导航切换后会重置，避免旧回复标题残留
- 对未加载缓存项执行跳转时，会进行定向滚动查找，找到对应 DOM 节点后再跳转
- 深度扫描会记录从顶部到下方发现的问题顺序，并在结束后重排目录
- 回复标题提取会统一合并语义标题与编号/视觉一级标题，次级标题继续挂在对应一级标题下
- 普通扫描后会后台预解析最多 12 条已加载回复；深度扫描和定位过程中不会预解析，避免滚动卡顿
- 页面加载和对话切换后会自动触发深度扫描（可在设置中关闭），扫描完成后恢复原滚动位置

## 隐私说明

`ChatGPT History Jump` 只在浏览器本地读取当前 ChatGPT 页面内容，用来生成页面内的问题目录和回复标题预览。扩展不会把提示词、回复、图片、账号信息或浏览活动上传到外部服务器。

扩展仅使用 `chrome.storage.local` 保存每个会话页面的侧边栏折叠状态和本地显示设置。完整说明见 [PRIVACY_POLICY.md](./PRIVACY_POLICY.md)。

## 当前限制

- 依赖 ChatGPT 当前页面 DOM 结构，若官方页面结构变动，识别逻辑可能需要调整
- 图片识别基于消息节点中的图片元素判断，不是对上传文件语义的深度理解
- 当前只处理“当前会话页面”的问题目录，不跨会话聚合
- 超长对话中，如果 ChatGPT 尚未把旧消息加载到 DOM，需要使用深度扫描或手动滚动后刷新
